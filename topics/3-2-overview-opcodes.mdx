## Categories of opcodes

1. **Constants**: Push values onto the stack (e.g., OP_0, OP_1, OP_16).

2. **Flow Control**: Conditionally execute code (e.g., OP_IF, OP_ELSE, OP_ENDIF).

3. **Stack**: Manipulate items on the stack (e.g., OP_DUP, OP_SWAP, OP_DROP).

4. **Splice**: Operations for string manipulation (e.g., OP_CAT, OP_SUBSTR, OP_SIZE).

5. **Bitwise Logic**: Perform bitwise operations (e.g., OP_AND, OP_OR, OP_XOR).

6. **Arithmetic**: Perform numeric operations (e.g., OP_ADD, OP_SUB, OP_MUL).

7. **Cryptographic**: Perform cryptographic operations (e.g., OP_RIPEMD160, OP_SHA1, OP_CHECKSIG).

8. **Locktime**: Deal with timelock features (e.g., OP_CHECKLOCKTIMEVERIFY).

## Key Characteristics

-   Opcodes are typically one byte in size.
-   Some opcodes have been disabled for security reasons.

    const OP_ADD = (stack) => {
        // 1. Check if there are at least 2 elements in the stack
        if (stack.length < 2) {
            return false; // Not enough elements in the stack
        }
        try {
            // 2. Pop the top two elements from the stack
            const b = stack.pop();
            const a = stack.pop();
            // 3. Add them together
            const result = a + b;
            // 4. Push the result back onto the stack
            stack.push(result);
            // 5. Return true as the operation was successful
            return true;
        } catch (error) {
            console.error("Error in OP_ADD:", error);
            return false; // Operation failed
        }
    };
